# akashic-sha512 — SHA-512 Cryptographic Hash

Full software FIPS 180-4 SHA-512 implementation.  Megapad-64's native
64-bit cells map directly to SHA-512's 64-bit word operations — no
masking or multi-cell arithmetic needed.

```forth
REQUIRE sha512.f
```

`PROVIDED akashic-sha512` — no additional dependencies.

## Design Principles

| Principle | Detail |
|---|---|
| **Pure software** | No hardware SHA-512 accelerator exists — full 80-round compression in Forth. |
| **64-bit native** | All SHA-512 word operations (add, rotate, shift, xor) use single-cell Forth ops.  `+` wraps mod 2^64 naturally.  No masking needed. |
| **Streaming** | `SHA512-BEGIN` / `SHA512-ADD` / `SHA512-END` — same pattern as SHA-256. |
| **One-shot convenience** | `SHA512-HASH` wraps the streaming API. |
| **VARIABLE-based state** | Module-scoped VARIABLEs for buffer pointer, byte count, message length, working variables, and loop temporaries.  Not re-entrant. |
| **Single-pass message length** | Tracks total bytes in a single 64-bit cell (sufficient for messages up to 2^64 − 1 bytes ≈ 16 EiB). |

## One-Shot Hashing

### SHA512-HASH

```forth
SHA512-HASH  ( src len dst -- )
```

Hash *len* bytes starting at *src* and write the 64-byte digest to *dst*.

```forth
CREATE msg  3 ALLOT  97 msg C!  98 msg 1+ C!  99 msg 2 + C!
CREATE out 64 ALLOT
msg 3 out SHA512-HASH
\ out = SHA-512("abc") = ddaf35a193617aba...
```

## Streaming API

### SHA512-BEGIN

```forth
SHA512-BEGIN  ( -- )
```

Initialize hash state from the eight IV constants.
Resets buffer position and message length counter.

### SHA512-ADD

```forth
SHA512-ADD  ( addr len -- )
```

Feed *len* bytes into the running hash.  Internally buffers up to
128 bytes (one block), compressing automatically when the buffer
fills.  May be called any number of times.

### SHA512-END

```forth
SHA512-END  ( dst -- )
```

Pad the message, append the 128-bit bit-length in big-endian,
compress the final block(s), and write the 64-byte digest to *dst*.

```forth
SHA512-BEGIN
header hdr-len SHA512-ADD
body   body-len SHA512-ADD
my-hash SHA512-END
```

## Hex Conversion & Display

### SHA512-.

```forth
SHA512-.  ( addr -- )
```

Print the 64-byte hash at *addr* as 128 lowercase hex characters.

### SHA512->HEX

```forth
SHA512->HEX  ( src dst -- n )
```

Convert the 64-byte hash at *src* to 128 lowercase hex characters
at *dst*.  Returns 128.

```forth
CREATE hbuf 64 ALLOT
CREATE hexbuf 128 ALLOT
msg 3 hbuf SHA512-HASH
hbuf hexbuf SHA512->HEX   \ returns 128
hexbuf SWAP TYPE           \ prints "ddaf35a1..."
```

## Comparison

### SHA512-COMPARE

```forth
SHA512-COMPARE  ( a b -- flag )
```

Constant-time comparison of two 64-byte hashes.
Returns `TRUE` (-1) if equal, `FALSE` (0) otherwise.

## Performance

SHA-512 is implemented entirely in software — each block requires
80 rounds of compression, each involving multiple 64-bit rotations,
shifts, and additions.

| Operation | Approximate cost |
|-----------|-----------------|
| One-block hash (≤ 128 bytes) | ~2 M emulator steps |
| Two-block hash (129–256 bytes) | ~4 M emulator steps |
| Streaming overhead per `SHA512-ADD` | Negligible (buffer copy) |

The primary consumer will be Ed25519 (pending), which requires
two SHA-512 calls per sign and three per verify.

## Internal Memory Layout

| Buffer | Size | Purpose |
|--------|------|---------|
| `_S512-K` | 640 bytes | 80 round constants (64-bit each) |
| `_S512-IV` | 64 bytes | 8 initial hash values |
| `_S512-H` | 64 bytes | Running hash state |
| `_S512-W` | 640 bytes | Message schedule (80 words) |
| `_S512-BUF` | 128 bytes | Input block buffer |

Total static allocation: ~1,536 bytes.
