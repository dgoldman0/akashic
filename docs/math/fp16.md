# akashic-fp16 — FP16 Tile-Engine Math for KDOS / Megapad-64

IEEE 754 half-precision (FP16) arithmetic using the Megapad-64 tile
engine as a scalar co-processor.  Each FP16 value lives as a raw
16-bit pattern on the 64-bit Forth data stack.

```forth
REQUIRE fp16.f
```

`PROVIDED akashic-fp16` — safe to include multiple times.

## Design Principles

| Principle | Detail |
|---|---|
| **Tile engine as scalar ALU** | Every binary op loads two FP16 values into lane 0 of 64-byte HBW tiles, fires one tile instruction, and reads lane 0 of the result tile. |
| **Lane 0 only** | The 32-lane SIMD width is not exploited here — that's the job of `fp16-batch.f` (Tier 4). |
| **Raw bit patterns** | Values are bare 16-bit integers on the stack. No boxing, no tagged types. |
| **Three scratch tiles** | `_FP16-TA` (src0), `_FP16-TB` (src1), `_FP16-TD` (dst) are allocated in HBW at load time. |
| **Not re-entrant** | Shared tile buffers mean concurrent callers would collide. |

## Arithmetic

All binary arithmetic words have the stack effect `( a b -- result )`
where `a`, `b`, and result are raw FP16 bit patterns.

| Word | Operation | Tile Instruction |
|---|---|---|
| `FP16-ADD` | $a + b$ | `TADD` |
| `FP16-SUB` | $a - b$ | `TSUB` |
| `FP16-MUL` | $a \times b$ | `TMUL` |
| `FP16-MIN` | $\min(a, b)$ | `TEMIN` |
| `FP16-MAX` | $\max(a, b)$ | `TEMAX` |

```forth
0x3C00 0x4000 FP16-ADD .  \ 1.0 + 2.0 → prints 16896 (0x4200 = 3.0)
```

## Unary Operations

Pure bit manipulation — no tile engine overhead.

| Word | Stack | Description |
|---|---|---|
| `FP16-NEG` | `( a -- -a )` | Flip sign bit (XOR 0x8000) |
| `FP16-ABS` | `( a -- \|a\| )` | Clear sign bit (AND 0x7FFF) |
| `FP16-SIGN` | `( a -- flag )` | 0 if positive, −1 if negative |

## Fused Multiply-Add

```forth
FP16-FMA  ( a b c -- a*b+c )
```

Uses the tile engine's `TFMA` instruction: loads `c` into the
destination tile, `a`/`b` into source tiles, then computes
$\text{dst}[i] = \text{src0}[i] \times \text{src1}[i] + \text{dst}[i]$
in a single fused operation with no intermediate rounding.

## Dot Product

### FP16-DOT

```forth
FP16-DOT  ( addr n -- acc )
```

Compute the dot product of `n` interleaved FP16 pairs starting at
`addr`.  Memory layout:

```
addr+0:  a0 (16-bit)
addr+2:  b0 (16-bit)
addr+4:  a1 (16-bit)
addr+6:  b1 (16-bit)
...
```

Each pair occupies 4 bytes.  Internally, pairs are de-interleaved
into separate tile buffers, 32 pairs at a time, and the tile engine's
`TDOT` instruction computes partial sums in the FP32 accumulator.
Partial sums are accumulated across chunks.

Returns the accumulator value (FP32-encoded in the low 32 bits of
a cell).

### FP16-DOT32

```forth
FP16-DOT32  ( addr-a addr-b -- acc )
```

Full-tile 32-pair dot product.  `addr-a` and `addr-b` must point
to 64-byte-aligned HBW tile buffers containing 32 FP16 values each.
Issues a single `TDOT` instruction and returns `ACC@`.

Use this when you've already arranged data in separate, aligned
tile buffers.

## Conversions

| Word | Stack | Description |
|---|---|---|
| `INT>FP16` | `( n -- fp16 )` | Convert signed integer to FP16. Clamped to ±65504. |
| `FP16>INT` | `( fp16 -- n )` | Truncate FP16 to signed integer (toward zero). |

```forth
42 INT>FP16 FP16>INT .   \ → 42
```

## Internals

| Word | Purpose |
|---|---|
| `_FP16-TA`, `_FP16-TB`, `_FP16-TD` | VARIABLEs holding HBW tile addresses (64 bytes each) |
| `_FP16-INIT-TILES` | Allocates and zeros the three tile buffers at load time |
| `_FP16-SETUP2` | `( a b -- )` Stores operands, sets FP16-MODE, assigns TSRC0/TSRC1/TDST |
| `_FP16-RESULT` | `( -- fp16 )` Reads lane 0 of the destination tile |
| `_DOT-TA`, `_DOT-TB` | Separate tile buffers used by FP16-DOT |
| `_DOT-DEINTERLEAVE` | `( addr n -- )` Splits interleaved pairs into two tiles |
