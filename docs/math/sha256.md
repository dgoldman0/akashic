# akashic-sha256 — SHA-256 Cryptographic Hash

Hardware-accelerated SHA-256 (FIPS 180-4) wrapper.  Delegates all
computation to the BIOS SHA-256 coprocessor at MMIO `0xFFFF_FF00_0000_0940`.

```forth
REQUIRE sha256.f
```

`PROVIDED akashic-sha256` — no additional dependencies.

## Design Principles

| Principle | Detail |
|---|---|
| **Hardware-accelerated** | All compression and padding is performed by the BIOS SHA-256 device — zero software round logic. |
| **Streaming** | `SHA256-BEGIN` / `SHA256-ADD` / `SHA256-END` allow incremental hashing of arbitrary-length messages without buffering the full input. |
| **One-shot convenience** | `SHA256-HASH` wraps the streaming API in a single call. |
| **Constant-time comparison** | `SHA256-COMPARE` uses OR-accumulation with no early exit. |
| **VARIABLE-based state** | Module-scoped VARIABLEs for the hex-conversion destination pointer.  Not re-entrant. |

```forth
SHA256-HEX-LEN  ( -- 64 )
```

Hex-encoded hash length in characters.

## One-Shot Hashing

### SHA256-HASH

```forth
SHA256-HASH  ( src len dst -- )
```

Hash *len* bytes starting at *src* and write the 32-byte digest to *dst*.

```forth
CREATE msg  3 ALLOT  97 msg C!  98 msg 1+ C!  99 msg 2 + C!
CREATE out 32 ALLOT
msg 3 out SHA256-HASH
\ out now contains SHA-256("abc") = ba7816bf...
```

## Streaming API

For messages that arrive in fragments or are too large to buffer.

### SHA256-BEGIN

```forth
SHA256-BEGIN  ( -- )
```

Reset the hardware hash state (sends SHA256-INIT to coprocessor).

### SHA256-ADD

```forth
SHA256-ADD  ( addr len -- )
```

Feed *len* bytes from *addr* into the running hash.  May be called
any number of times between `SHA256-BEGIN` and `SHA256-END`.

### SHA256-END

```forth
SHA256-END  ( dst -- )
```

Finalize the hash and write the 32-byte digest to *dst*.

```forth
SHA256-BEGIN
part1 n1 SHA256-ADD
part2 n2 SHA256-ADD
my-hash SHA256-END
\ my-hash = SHA-256( part1 || part2 )
```

## Hex Conversion & Display

### SHA256-.

```forth
SHA256-.  ( addr -- )
```

Print the 32-byte hash at *addr* as 64 lowercase hex characters
to the current output device.

### SHA256->HEX

```forth
SHA256->HEX  ( src dst -- n )
```

Convert the 32-byte hash at *src* to 64 lowercase hex characters
at *dst*.  Returns 64 (the character count).

```forth
CREATE hbuf 32 ALLOT
CREATE hexbuf 64 ALLOT
msg 3 hbuf SHA256-HASH
hbuf hexbuf SHA256->HEX    \ returns 64
hexbuf SWAP TYPE            \ prints "ba7816bf..."
```

## Comparison

### SHA256-COMPARE

```forth
SHA256-COMPARE  ( a b -- flag )
```

Constant-time comparison of two 32-byte hashes.
Returns `TRUE` (-1) if equal, `FALSE` (0) otherwise.

Uses OR-accumulation over all 32 bytes — timing does not depend
on the position of the first differing byte.

## Hardware Detail

The BIOS SHA-256 coprocessor lives at MMIO base `0xFFFF_FF00_0000_0940`:

| Offset | Register | Description |
|--------|----------|-------------|
| `+0x00` | CMD | 1 = INIT, 3 = FINAL |
| `+0x08` | STATUS | Busy / done flags |
| `+0x10` | DIN | Byte input (auto-compresses at 64 bytes) |
| `+0x18..+0x37` | DOUT | 32-byte hash output (big-endian) |

The Akashic wrapper calls BIOS words `SHA256-INIT`, `SHA256-UPDATE`,
and `SHA256-FINAL` which manage these registers.
