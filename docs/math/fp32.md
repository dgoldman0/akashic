# akashic-fp32 — Software IEEE 754 Binary32 for KDOS / Megapad-64

Full software implementation of single-precision (FP32) floating-point
arithmetic using only the Megapad-64's 64-bit integer ALU.  Intended
for final-stage statistical computations where FP16's 3.3 decimal
digits are insufficient — variance, regression coefficients, R² scores,
etc.

```forth
REQUIRE fp32.f
```

`PROVIDED akashic-fp32` — safe to include multiple times.

## Design Principles

| Principle | Detail |
|---|---|
| **Pure integer emulation** | Every operation is implemented with 64-bit integer shifts, adds, and multiplies — no hardware FP unit required. |
| **IEEE 754 compliant** | Handles ±0, ±∞, NaN, subnormals (flushed to zero), and uses round-to-nearest-even. |
| **Raw bit patterns** | FP32 values live as bare 32-bit integers in the low half of a 64-bit Forth cell. |
| **Mixed-precision pipeline** | Designed to sit between FP16 tile-engine bulk processing and 48.16 integer accumulators. |
| **VARIABLE-based state** | Internal temporaries use module-scoped VARIABLEs (no locals). Not re-entrant. |

## Classification

Predicate words return a Forth flag (0 = false, −1 = true).

| Word | Stack | Description |
|---|---|---|
| `_FP32-INF?` | `( fp32 -- flag )` | True if ±∞ |
| `_FP32-NAN?` | `( fp32 -- flag )` | True if NaN (any payload) |
| `_FP32-ZERO?` | `( fp32 -- flag )` | True if +0 or −0 |
| `FP32-0=` | `( fp32 -- flag )` | Same as `_FP32-ZERO?` |

## Comparison

Comparisons convert sign-magnitude bit patterns to a sortable integer key, then
use ordinary integer comparison.  NaN propagation follows IEEE semantics (NaN ≠
everything).

| Word | Stack | Description |
|---|---|---|
| `FP32<` | `( a b -- flag )` | $a < b$ |
| `FP32>` | `( a b -- flag )` | $a > b$ |
| `FP32=` | `( a b -- flag )` | $a = b$ |
| `FP32<=` | `( a b -- flag )` | $a \le b$ |
| `FP32>=` | `( a b -- flag )` | $a \ge b$ |

```forth
FP32-ONE FP32-TWO FP32< .   \ → -1  (true)
```

## Arithmetic

Binary operations: `( a b -- result )`.  All handle ±0, ±∞, and NaN.

| Word | Operation | Notes |
|---|---|---|
| `FP32-ADD` | $a + b$ | Mantissa alignment, 3 guard bits, round-to-nearest-even |
| `FP32-SUB` | $a - b$ | Negate `b`, then ADD |
| `FP32-MUL` | $a \times b$ | 24 × 24 → 48-bit product, single rounding |
| `FP32-DIV` | $a \div b$ | Long division with 23-bit quotient |

Unary operations:

| Word | Stack | Description |
|---|---|---|
| `FP32-NEGATE` | `( a -- -a )` | Flip sign bit (XOR bit 31) |
| `FP32-ABS` | `( a -- \|a\| )` | Clear sign bit (AND 0x7FFFFFFF) |

```forth
FP32-ONE FP32-TWO FP32-ADD .   \ → 1077936128 (0x40400000 = 3.0)
FP32-TEN FP32-TWO FP32-DIV .   \ → 1084227584 (0x40A00000 = 5.0)
```

## Fused Multiply-Add

```forth
FP32-FMA  ( a b c -- a*b+c )
```

Computes $a \times b + c$.  Implemented as `MUL` then `ADD` (two
rounding steps), which is acceptable for statistics pipelines
where the error budget is dominated by FP16 input quantisation.

## Approximations

### FP32-RECIP

```forth
FP32-RECIP  ( a -- 1/a )
```

Reciprocal via Newton–Raphson iteration.  Uses magic constant
`0x7EF127EA` for the initial estimate, then 3 refinement steps:
$x_{n+1} = x_n \cdot (2 - a \cdot x_n)$.  Accurate to ±1 ULP
for normal inputs.

### FP32-SQRT

```forth
FP32-SQRT  ( a -- √a )
```

Square root via Newton–Raphson.  Initial estimate uses
$(x \gg 1) + \mathtt{0x1FBB4000}$, then 4 iterations of
$s' = \frac{1}{2}(s + x / s)$.  Accurate to ±1 ULP for normal
positive inputs.  Returns NaN for negative inputs, +0 for +0,
+∞ for +∞.

## Conversions

### FP16 ↔ FP32

| Word | Stack | Description |
|---|---|---|
| `FP16>FP32` | `( fp16 -- fp32 )` | Expand: sign ≪ 16, exp + 112, frac ≪ 13. Handles ±0, inf, NaN. |
| `FP32>FP16` | `( fp32 -- fp16 )` | Narrow with round-to-nearest-even on the 13 dropped mantissa bits. |

```forth
0x3C00 FP16>FP32 .   \ FP16 1.0 → 1065353216 (FP32 1.0)
```

### Integer ↔ FP32

| Word | Stack | Description |
|---|---|---|
| `INT>FP32` | `( n -- fp32 )` | Signed integer to FP32 (finds MSB, packs exponent). |
| `FP32>INT` | `( fp32 -- n )` | FP32 to signed integer (truncation toward zero). |

### Fixed-Point ↔ FP32

| Word | Stack | Description |
|---|---|---|
| `FX>FP32` | `( fx -- fp32 )` | 8.8 fixed-point to FP32 (treats as integer/256). |
| `FP32>FX` | `( fp32 -- fx )` | FP32 to 8.8 fixed-point. |

### Tile Accumulator

| Word | Stack | Description |
|---|---|---|
| `ACC>FP32` | `( -- fp32 )` | Read tile engine ACC@ (CSR 0x19), mask to 32 bits. |

## Internals

### Field Extraction

| Word | Stack | Description |
|---|---|---|
| `_FP32-SIGN` | `( fp32 -- 0\|1 )` | Bit 31 |
| `_FP32-EXP` | `( fp32 -- exp )` | Bits 30–23 (biased exponent, 0–255) |
| `_FP32-FRAC` | `( fp32 -- frac )` | Bits 22–0 (mantissa without implicit 1) |
| `_FP32-PACK` | `( sign exp frac -- fp32 )` | Assemble from components |

### Sort Key

```forth
_FP32-SORTKEY  ( fp32 -- key )
```

Converts sign-magnitude IEEE 754 encoding to a two's-complement
sortable integer.  Positive values map as-is; negative values are
bit-flipped.  Enables comparison via ordinary `<` / `>`.

### Module Variables

| Variable | Purpose |
|---|---|
| `_FM-SA`, `_FM-SB` | Signs of operands a, b |
| `_FM-EA`, `_FM-EB` | Biased exponents of a, b |
| `_FM-MA`, `_FM-MB` | Mantissas (with implicit 1) |
| `_FM-TMP` | General temporary |
| `_FM-GRD` | Guard bits for rounding |
