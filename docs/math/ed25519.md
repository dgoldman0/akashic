# akashic-ed25519 — Ed25519 Digital Signatures (RFC 8032)

Ed25519 key generation, signing, and verification over the twisted
Edwards curve -x² + y² = 1 + d·x²·y² mod p = 2²⁵⁵ − 19.
Targets the Megapad-64 Field ALU for all 256-bit modular arithmetic.

```forth
REQUIRE ed25519.f
```

`PROVIDED akashic-ed25519` — depends on `akashic-sha512`, `akashic-field`.

## Design Principles

| Principle | Realisation |
|---|---|
| **Hardware-accelerated** | All 256-bit field ops (mod p and mod L) go through the MMIO Field ALU |
| **Pure Forth** | No assembly; only MMIO through `field.f` |
| **RFC 8032 compliant** | Matches all three official test vectors (empty, 1-byte, 2-byte messages) |
| **Single-alloc buffers** | All scratch space is statically allocated (no heap) |
| **Constant-time** | Scalar multiply uses branchless double-and-always-add with arithmetic CMOV — no secret-dependent branches |
| **Malleable-sig resistant** | `ED25519-VERIFY` rejects S ≥ L (RFC 8032 §5.2.7 check) |
| **Canonical decoding** | Point decode rejects non-canonical y ≥ p |
| **Secret zeroization** | Scratch scalars, hash buffers, and point accumulators are zeroed after sign/verify |
| **Not reentrant** | Uses module-level VARIABLEs; wrapped with `GUARD`/`WITH-GUARD` for concurrency safety |

## Key Generation

### ED25519-KEYGEN

```forth
ED25519-KEYGEN  ( seed pub priv -- )
```

Derive a keypair from a 32-byte random seed.

| Parameter | Size | Description |
|---|---|---|
| `seed` | 32 bytes | Random secret (caller-provided) |
| `pub` | 32 bytes | Output public key (compressed Edwards point) |
| `priv` | 64 bytes | Output private key (clamped scalar ∥ prefix) |

**Algorithm:**
1. `h = SHA-512(seed)` → 64 bytes
2. Clamp `h[0..31]` → scalar `a` (clear low 3 bits, clear high bit, set bit 254)
3. `priv = a ∥ h[32..63]`
4. `pub = encode(a · B)` where B is the base point

## Signing

### ED25519-SIGN

```forth
ED25519-SIGN  ( msg len priv pub sig -- )
```

Sign a message.

| Parameter | Size | Description |
|---|---|---|
| `msg` | `len` bytes | Message to sign (addr 0 + len 0 for empty) |
| `priv` | 64 bytes | Private key from `ED25519-KEYGEN` |
| `pub` | 32 bytes | Public key from `ED25519-KEYGEN` |
| `sig` | 64 bytes | Output signature |

**Algorithm (RFC 8032 §5.1.6):**
1. `r = SHA-512(prefix ∥ msg) mod L`
2. `R = encode(r · B)`
3. `h = SHA-512(R ∥ pub ∥ msg) mod L`
4. `S = (r + h · a) mod L`
5. `sig = R ∥ S`

## Verification

### ED25519-VERIFY

```forth
ED25519-VERIFY  ( msg len pub sig -- flag )
```

Verify a signature. Returns TRUE (-1) on success, FALSE (0) on failure.

| Parameter | Size | Description |
|---|---|---|
| `msg` | `len` bytes | Original message |
| `pub` | 32 bytes | Signer's public key |
| `sig` | 64 bytes | Signature to verify |

**Algorithm (RFC 8032 §5.1.7):**
1. Decode `A` from `pub` (reject non-canonical y ≥ p)
2. Decode `R` from `sig[0..31]` (reject non-canonical y ≥ p)
3. Reject if S ≥ L (malleability check)
4. `h = SHA-512(R ∥ pub ∥ msg) mod L`
5. Check: `encode(S · B) == encode(R + h · A)`
6. Zeroize scratch buffers (`_ED-H64`, `_ED-SC1`)

## Usage Example

```forth
REQUIRE akashic/math/ed25519.f

\ Allocate buffers
CREATE MY-SEED 32 ALLOT
CREATE MY-PUB  32 ALLOT
CREATE MY-PRIV 64 ALLOT
CREATE MY-SIG  64 ALLOT
CREATE MY-MSG   5 ALLOT

\ Fill seed with random bytes (application-specific)
\ ... fill MY-SEED ...

\ Generate keypair
MY-SEED MY-PUB MY-PRIV ED25519-KEYGEN

\ Sign a 5-byte message
MY-MSG 5 MY-PRIV MY-PUB MY-SIG ED25519-SIGN

\ Verify
MY-MSG 5 MY-PUB MY-SIG ED25519-VERIFY
IF ." Signature valid" THEN
```

## Internals

### Curve Parameters

| Symbol | Value | Buffer |
|---|---|---|
| p | 2²⁵⁵ − 19 | `_ED-P` (32-byte LE), also `FIELD-USE-25519` |
| L | 2²⁵² + 27742317777372353535851937790883648493 | `_ED-L` |
| d | −121665/121666 mod p | `_ED-D` |
| B | (Bx, By) on the curve | `_ED-BX`, `_ED-BY` |
| √(−1) | mod p | `_ED-SQRTM1` |
| (p+3)/8 | Exponent for candidate square root | `_ED-EXP38` |

### Point Representation

Extended coordinates (X, Y, Z, T) where x = X/Z, y = Y/Z, T = X·Y/Z.
Each point occupies 128 bytes (4 × 32-byte field elements).

### Point Operations

| Word | Formula | Reference |
|---|---|---|
| `_ED-DBL` | dbl-2008-hwcd (a = −1) | EFD |
| `_ED-ADD` | add-2008-hwcd-4 (a = −1) | EFD |
| `_ED-SMUL` | Constant-time double-and-always-add + CMOV | Standard |

### Hash-to-Scalar Reduction

`_ED-REDUCE ( hash64 result -- )` converts a 64-byte SHA-512 output to a
scalar mod L:

1. Split into `lo` (bytes 0–31) and `hi` (bytes 32–63)
2. `reduced_hi = hi × R₂₅₆ mod L` where R₂₅₆ = 2²⁵⁶ mod L
3. `reduced_lo = lo × 1 mod L` (ensures lo < L via FIELD-MUL)
4. `result = reduced_lo + reduced_hi mod L`

The explicit reduction of `lo` via `FIELD-MUL` is required because the
hardware `FIELD-ADD` performs only a single conditional subtraction, which
is insufficient when `lo` (a raw 256-bit hash chunk) exceeds L ≈ 2²⁵².

### Prime Switching

The module switches between two field primes:
- `_ED-USE-P` → p = 2²⁵⁵ − 19 for point arithmetic
- `_ED-USE-L` → L (group order) with pinv=0 (schoolbook multiply) for scalar arithmetic

### Scratch Buffers

| Buffer | Size | Purpose |
|---|---|---|
| `_ED-T1`–`_ED-T8` | 32 bytes each | Temporary field elements |
| `_ED-PA`–`_ED-PD` | 128 bytes each | Extended points |
| `_ED-SC1`–`_ED-SC3` | 32 bytes each | Scalars (r, h, S) |
| `_ED-H64` | 64 bytes | SHA-512 hash output |
| `_ED-ENC1`, `_ED-ENC2` | 32 bytes each | Encoded points for comparison |
| `_ED-P` | 32 bytes | Curve25519 prime p (LE), for canonical y check |
| `_ED-CT-MSK` | 1 cell | Arithmetic mask for `_ED-CT-SELECT` |

### Security Helpers

| Word | Stack | Description |
|---|---|---|
| `_ED-CT-SELECT` | `( dst src len bit -- )` | Branchless conditional copy: copies `src` to `dst` when `bit=1`, no-op when `bit=0`. Uses arithmetic mask (NEGATE + AND/XOR), no branches. |
| `_ED-BYTES-GTE?` | `( a b -- flag )` | 32-byte unsigned LE comparison: TRUE if `a >= b`. Used for canonical y and S < L checks. |
| `_ED-BYTES-LT?` | `( a b -- flag )` | 32-byte unsigned LE comparison: TRUE if `a < b`. |

## Test Coverage

25 tests across 3 RFC 8032 test vectors and 4 hardening categories:

**Functional (15 tests):**
- Key generation (3 vectors)
- Signature generation (3 vectors, exact match)
- Signature verification (3 vectors)
- Reject corrupted signature
- Reject wrong public key
- Constants
- Module compilation

**Hardening (10 tests):**
- P01 Constant-time SMUL: 1·B = B, 0·B = identity
- P02 Malleability: original passes, S=L rejected, S=0 passes guard
- P04 Canonical y: y=0xFF…FF rejected, y=p rejected, y=p−1 passes guard
- P05 Zeroization: `_ED-H64`, `_ED-SC1`, `_ED-PA` all zeroed after sign

Source: ~453 lines of Forth.  Test file: `local_testing/test_ed25519.py`.
